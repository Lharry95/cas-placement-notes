# Week 2 - Monday 27th April - 3rd May 2026

## What I worked on

- Focused on improving and restructuring the UI of an existing IFC Parser & 3D Visualization tool
- Worked within an existing codebase rather than building from scratch
- Redesigned the layout into a clearer three-column structure:
  - left tab navigation (icon-based)
  - central content panel
  - right-side detail panel
- Updated and organised tab-based navigation (General, 3D View, Entities, Properties, etc.)
- Improved the file upload UI with a cleaner header and custom upload button
- Added and structured UI components such as:
  - upload overlay and drag-and-drop area
  - progress/loading states
  - filter controls and action buttons
- Connected UI elements to existing JavaScript functions (e.g. `switchTab`, `togglePanel`)

## What I learned

- How to work within an existing codebase and adapt it without breaking functionality
- How UI structure impacts usability and clarity in more complex applications
- How to organise large interfaces using sections like `header`, `nav`, `main`, and `aside`
- How HTML interacts with existing JavaScript logic through event handlers
- How AI tools can assist with layout ideas, debugging, and speeding up development

## Challenges

- Understanding the existing structure before making UI changes
- Avoiding breaking existing functionality while updating layout and design
- Keeping the UI consistent across many different sections and tabs
- Managing a large number of elements and ensuring everything remained organised

## Solutions

- Made incremental changes rather than large rewrites
- Reused existing structure where possible instead of replacing everything
- Grouped related features into clearly defined sections (tabs and panels)
- Used consistent naming and layout patterns to improve readability

## Key takeaways

- Working in an existing codebase requires understanding before changing
- Good UI structure makes complex tools easier to use and navigate
- Small, incremental improvements are safer and more effective than large changes
- AI can be a useful tool for learning and problem-solving when used thoughtfully

## Reflection

- This week helped me understand what it’s like to contribute to a real project rather than starting from scratch
- I became more confident working with existing code instead of relying only on my own builds
- I want to continue improving how I structure UI and connect it more cleanly with JavaScript
